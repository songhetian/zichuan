## 资产管理系统 UI/UX 改进计划

### 1. Dashboard 首页（新增）

**现状**：登录后直接跳 `/assets`，无全局概览。

**改进**：新增 `/dashboard` 首页，上半部分概览卡片（设备总数、闲置、在用、维修中、报废）+ 分类分布饼图 + 库存柱状图，下半部分待办任务列表（待分配设备、维修中设备、库存不足配件、保修即将到期设备）+ 最近操作时间线。

**影响文件**：新增 `src/app/(main)/page.tsx`、`src/app/(main)/dashboard-client.tsx`、新增统计相关 Server Action。

---

### 2. 设备管理 — 批量操作（增强）

**现状**：只能逐台操作（分配/归还/送修/报废），无多选。

**改进**：DataTable 行前加复选框，选中后顶部出现批量操作栏（批量分配、批量归还、批量报废），支持全选当前页。批量分配弹窗支持选择目标员工。

**影响文件**：`asset-list-client.tsx`、`data-table.tsx`（增加 rowSelection）、`lifecycle.actions.ts`（已支持数组，无需改后端）。

---

### 3. 分类管理 — 树形表格 + 拖拽排序（重构）

**现状**：设备分类和配件分类都是扁平 DataTable，两级层级关系不可见。创建分类无父分类选择。

**改进**：改为树形表格（Tree Table），父行可展开/收起，子分类缩进显示。支持拖拽调整排序和层级。创建表单增加"所属父分类"下拉选择。

**影响文件**：`asset-categories-client.tsx`、`component-categories-client.tsx`、新增 `tree-table.tsx` 组件、新增排序相关 Server Action。

---

### 4. 配件库存 — 手动入库操作（新增）

**现状**：库存页面只展示流水日志，无手动入库入口。库存变动只能由设备操作自动触发。

**改进**：库存页面顶部增加"采购入库"按钮，弹窗选择配件型号 + 输入数量 + 备注，提交后创建 ComponentStockLog 记录并更新库存。库存列表从流水页拆分出来作为主视图。

**影响文件**：`stock-log-client.tsx` 或新增 `stock-client.tsx`、新增入库 Server Action。

---

### 5. 员工管理 — 行内展开设备面板（增强）

**现状**：员工列表"在用设备数"列硬编码 `"-"`，无法查看员工名下设备。

**改进**：修复 assetCount 列显示真实数据。点击员工行可展开 Inline 面板，展示名下所有设备表格 + 分配历史。

**影响文件**：`employee-list-client.tsx`、`employee.actions.ts`（增加查询员工设备接口）。

---

### 6. 模板 BOM — 侧滑面板 + 拖拽 + 复制（重构）

**现状**：BOM 编辑在 Dialog 弹窗内的表格里，配件多时体验差，无拖拽排序，无模板间复制。

**改进**：编辑改为全屏侧滑 Sheet（Slide-over），BOM 行可拖拽排序，增加"从其他模板复制配件"按钮。配件添加支持搜索和批量选择。

**影响文件**：`template-list-client.tsx`、新增 `sheet.tsx` UI 组件（或复用 shadcn Sheet）。

---

### 7. 盘点 — 卡片式逐项盘点界面（重构）

**现状**：创建盘点任务后只有"完成"按钮，缺少逐项盘点操作。statusFilter 未传递到后端。

**改进**：盘点任务创建后进入盘点操作界面——卡片式检查清单，每台设备一张卡片，管理员点选"正常/盘亏/盘盈" + 输入备注，支持键盘快捷键（N=正常、M=盘亏、E=盘盈）。底部进度条显示已盘/未盘数量。

**影响文件**：`stocktake-list-client.tsx`、新增 `stocktake-detail-client.tsx`、`stocktake.actions.ts`（增加逐项更新接口）。

---

### 8. 全局搜索 — 搜索框 + Command Palette（新增）

**现状**：无全局搜索，每个页面独立筛选。

**改进**：Header 增加全局搜索框（轻量搜索，输入即搜设备/员工/模板），Ctrl+K 唤起 Command Palette（深度导航，支持搜索页面、快速跳转、执行常用操作如"新建设备"）。

**影响文件**：`header.tsx`、新增 `command-palette.tsx`、新增全局搜索 Server Action。

---

### 9. 设备详情 — 生命周期时间线（增强）

**现状**：操作记录 tab 是纯表格流水账。

**改进**：改为垂直时间线组件（Timeline），每个节点带状态颜色标识（绿色=分配、蓝色=归还、黄色=维修、红色=报废）和对应图标，时间轴左侧显示时间，右侧显示操作详情。

**影响文件**：`asset-detail-client.tsx`、新增 `timeline.tsx` 组件。

---

### 10. 表单实时校验（全局增强）

**现状**：所有表单"点提交后才验证"，后端报错需重新填写。

**改进**：所有表单统一接入 React Hook Form + Zod Schema 校验，实时提示（工号重复、邮箱格式、必填项等），提交前完成所有校验。

**影响文件**：所有含表单的 `*-client.tsx` 文件。

---

### 11. 加载骨架屏（全局增强）

**现状**：页面加载时无视觉反馈或空白闪烁。

**改进**：每个页面增加 `loading.tsx`，使用 Skeleton 骨架屏组件模拟页面结构，提供平滑的加载过渡。

**影响文件**：每个路由目录新增 `loading.tsx`。

---

### 12. 导出预览（增强）

**现状**：点"导出 Excel"直接下载，无法预览或选择字段。

**改进**：导出前弹出预览对话框，展示将要导出的数据摘要和字段列表，支持勾选/取消字段，确认后生成下载。

**影响文件**：`asset-list-client.tsx`、`employee-list-client.tsx`、新增 `export-preview.tsx` 组件。
