# 设备生成模式：按模板分配时生成实体

设备实体在分配给员工时才生成（从设备模板创建），而不是提前批量入库。生成时扣减对应配件库存。设备一旦生成即为独立实体，拥有完整生命周期履历，归还后进入设备池（闲置状态），可再次分配。

**原因**：
- 用户明确选择 B 方案：组合成的设备只是通用型号，分配给具体的人之后才有具体实体
- 与配件库存管理联动更紧密：配件 → 模板 → 生成设备 → 扣减库存，链路清晰
- 设备池概念解决了升级后型号与配置不一致的问题：型号只是出生标签，实际配置以设备当前配件为准

**Consequences**：
- 没有"型号库存"的概念，只有配件库存和设备池（按状态筛选）
- 分配有两种方式：从闲置设备池选具体设备，或按模板生成新设备直接分配
- 升级不改变设备的模板归属，但实际配置以当前配件为准
- 批量采购时通过"按模板批量生成闲置设备"来操作
